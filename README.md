# Skills 技能集

本目录包含一系列用于提升前端开发效率的 CatPaw 技能，覆盖环境搭建、需求分析、技术设计、代码评审等完整开发流程。

## 📋 技能清单

### 1. frontend-env-installer (前端环境一键安装)

**功能概述**：一键安装完整的前端开发环境，支持多平台（macOS/Linux/Windows）。

**核心能力**：
- ✅ 自动检测操作系统
- ✅ 安装 Node.js（支持指定版本）
- ✅ 安装包管理器（npm/yarn/pnpm）
- ✅ 配置 npm 镜像源（淘宝镜像等）
- ✅ 安装常用全局工具（Vue CLI、Create React App、Vite 等）
- ✅ 验证安装结果

**触发关键词**：
- 安装前端环境
- 配置前端开发环境
- 初始化前端环境
- 搭建前端开发环境
- 安装 Node.js
- 配置 npm/yarn/pnpm
- 前端环境初始化

- ✅ 自动分离前端和后端开发任务
- ✅ 识别接口定义和数据交互
- ✅ 识别页面/组件开发需求
- ✅ 识别疑问点和争议内容
- ✅ 生成 Markdown/Excel 格式分工表格

**触发关键词**：
- 需求分析
- 前后端分工
- 分离前后端
- 需求拆解
- 分析需求文档
- 需求文档拆分
- 前后端任务分离
- 解析需求文档
- 拆分需求

### 4. full-stack-development-pipeline (全栈前端开发流水线)

**功能概述**：一站式完成从需求分析到项目开发的全流程，自动化串联三个核心阶段。

**三阶段流程**：
1. **需求分析阶段**：使用 requirement-splitter 分析需求文档并拆分前后端任务
2. **技术设计阶段**：使用 frontend-tech-doc-generator 生成完整技术文档
3. **项目开发阶段**：在具体项目中执行开发实施

**核心价值**：
- 🔄 流程自动化：三个阶段无缝衔接，自动流转
- 📋 输出标准化：每个阶段产生标准化的输出产物
- 🎯 前后端协同：清晰的任务分工，明确的责任边界
- 📖 文档完备：技术文档齐全，便于团队协作和后续维护
- ⚡ 快速启动：从需求到开发，一键式完成准备

**适用场景**：
- 有需求文档的新项目开发
- 企业级应用前端开发
- 团队协作的前端项目
- 需要规范化输出的开发任务

**触发关键词**：
- 全流程开发
- 需求到上线
- 完整开发流程
- 需求文档开发
- 端到端开发
- 需求分析+技术设计+开发
- 一键开发全流程

- ⚡ 快速启动：避免过度规划，只做必要准备

**项目类型支持**：

**项目规模适配**：

- 大型项目：完整流程（详细规划）

**触发关键词**：

### 6. git-diff-reviewer (Git Diff 代码评审)

**功能概述**：查看当前项目 git diff 并自动进行代码评审，基于代码规范、最佳实践、潜在问题等维度进行智能评审。

**核心特点**：
- 🔍 多模式 Diff 查看：支持工作区、暂存区、分支间、提交间的差异查看
- 🤖 智能代码评审：自动识别代码问题并提供改进建议
- 📊 多维度分析：代码规范、安全性、性能、可维护性等
- 🎯 项目级执行：仅在当前项目目录下运行，避免误操作
- 💡 可操作建议：提供具体的代码修改建议和最佳实践

**支持的 Diff 模式**：
- 工作区变更（git diff）
- 暂存区变更（git diff --cached）
- 所有变更（git diff HEAD）
- 分支间差异（git diff <branch1> <branch2>）
- 提交间差异（git diff <commit1> <commit2>）
- 最近 N 次提交的变更（git diff HEAD~N HEAD）

**评审维度**：
1. 代码规范（命名规范、代码格式、注释规范等）
2. 最佳实践（设计原则、类型安全、现代语法等）
3. 潜在问题（类型错误、空值处理、内存泄漏等）
4. 性能优化（算法优化、函数优化、渲染优化等）
5. 安全问题（XSS 风险、敏感信息、权限控制等）
6. 可维护性（代码可读性、模块化、测试覆盖等）

**触发关键词**：
- 代码评审
- code review
- 查看变更
- git diff
- 检查代码
- 评审代码
- review代码
- 代码审查

## 📝 最佳实践

### 1. 环境准备
首次使用建议先运行 `frontend-env-installer` 确保开发环境完备。

### 2. 需求驱动开发
有需求文档时，使用 `full-stack-development-pipeline` 可以一站式完成从需求分析到开发的全流程。

### 3. 快速启动
不确定从哪里开始时，使用 `one-stop-development`，它会根据项目情况智能推荐步骤。

### 4. 持续代码质量
在开发过程中，定期使用 `git-diff-reviewer` 进行代码评审，及时发现问题。

### 5. 文档按需生成
- 小型项目：无需生成详细文档，直接开发
- 中型项目：生成精简版技术文档
- 大型项目：生成完整版技术文档
